# Codex Operating Instructions

## Project Role

You are helping build `Fourteen Hours Apart`, a long-form English novel project for KDP. Treat the repository like a software project: preserve structure, maintain continuity, create reusable templates, and make careful incremental changes.

## First Principles

- This is fiction, not memoir.
- Preserve emotional truth, not literal fact.
- Do not use real names.
- Do not use exact job titles, institutions, private conversations, or identifying details when they create privacy risk.
- Write for English-language romance and romantic fiction readers first.
- Do not draft full chapters unless explicitly asked and the outline supports it.

## Required Workflow For New Episodes

When the author provides a new episode:

1. Create a cleaned raw episode file in `01_raw_episodes/`.
2. Transform it into a scene card in `02_scene_cards/`.
3. Suggest possible placement in `03_outline/chapter_outline.md`.
4. Add privacy-risk notes.
5. Suggest fictionalized scene material only when useful.

## Style Rules

- Use close third person.
- Alternate Daniel and Grace only when the scene benefits from it.
- Keep the tone mature, restrained, emotionally precise, literary but readable.
- Let dialogue carry subtext.
- Show cultural difference through choices, silences, habits, obligations, and misunderstandings.
- Avoid explanation dumps, self-pity, melodrama, and memoir-like justification.

## Repository Hygiene

- Keep files in Markdown.
- Prefer concise, structured updates over sprawling notes.
- Update project-bible files when a new rule, timeline fact, or character truth becomes canonical.
- Keep placeholders clearly marked as placeholders.
- Do not delete prior author material unless explicitly asked.
